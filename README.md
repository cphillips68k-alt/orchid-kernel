## About this project

Orchid is a microkernel designed to be simple, understandable, and completely self-contained. The code in this repository was generated by an AI assistant, working from detailed human specifications, architectural requirements, and design constraints. Every line of code exists because a person decided it should be there and what it should do—the AI acted as a translator from design to implementation.

## Why this exists

The original Rust version worked but pulled in 80+ crates for basic functionality like serial output and VGA text. This rewrite does everything with:

- A single vendored Limine 8.x protocol header (0BSD licensed, from the Limine project)
- The C standard library's freestanding headers (no host OS dependencies)
- Raw inline assembly where needed
- A bootloader (Limine) that is automatically vendored at build time from the official binary branch

No crates. No npm-style dependency tree. No build system beyond a Makefile. No runtime dependencies beyond what the bootloader provides.

## Building

Requirements:

- GCC cross-compiler targeting `x86_64-elf`
- GNU binutils for the same target
- QEMU for testing
- `git` (to automatically download the Limine bootloader binary release)
- `xorriso` (to create the bootable ISO image)

chmod +x run_qemu.sh
./run_qemu.sh
